Git e GitHub

O que é o Git e o GitHub?

O Git é um sistema de controlo de versões que permite acompanhar as alterações nos ficheiros ao longo do tempo. Imagina que é como ter uma máquina do tempo para o teu código!

Porquê usar Git?
• Guardar diferentes versões do teu projeto
• Colaborar com outras pessoas sem conflitos
• Recuperar versões anteriores se algo correr mal
• Trabalhar em diferentes funcionalidades ao mesmo tempo

O GitHub é uma plataforma online que aloja repositórios Git. É como uma rede social para programadores, onde podes partilhar os teus projetos e colaborar com outros.

O Git trabalha localmente. O GitHub guarda tudo online!

Conceitos Fundamentais

1 Repositório (Repository)

É a pasta do teu projeto que está a ser monitorizada pelo Git. Contém todos os ficheiros e o histórico de alterações.

2 Commit

É como tirar uma fotografia do teu projeto num determinado momento. Cada commit guarda as alterações que fizeste.

3 Branch (Ramificação)

É uma linha independente de desenvolvimento. Podes trabalhar em funcionalidades novas sem afetar o código principal.

4 Clone

Fazer uma cópia de um repositório do GitHub para o teu computador.

5 Push

Enviar as tuas alterações locais para o GitHub.

6 Pull

Trazer alterações do GitHub para o teu computador.

Comandos Essenciais do Git

Iniciar um Repositório

git init

Transforma uma pasta normal num repositório Git.

Verificar o Estado

git status

Mostra quais ficheiros foram modificados, adicionados ou eliminados.

Adicionar Ficheiros

git add nome_do_ficheiro

Prepara um ficheiro específico para o próximo commit.

git add .

Adiciona TODOS os ficheiros modificados de uma vez.

Fazer um Commit

git commit -m "Mensagem descritiva do que foi alterado"

Guarda as alterações com uma mensagem explicativa.

Boas Práticas para Mensagens de Commit:
• Usa frases curtas e descritivas
• Começa com um verbo no imperativo (ex: "Adiciona", "Corrige", "Atualiza")
• Exemplo: "Adiciona função de login" em vez de "fiz coisas"

Ver Histórico

git log

Mostra o histórico de todos os commits realizados.

Criar uma Ramificação

git branch nome_do_branch

Cria uma nova ramificação de desenvolvimento.

Mudar de Ramificação

git checkout nome_do_branch

Muda para outra ramificação de trabalho.

git checkout -b nome_do_branch

Cria uma nova ramificação e muda para ela ao mesmo tempo.

Enviar para o GitHub

git push origin main

Envia os commits da ramificação "main" para o GitHub.

Trazer do GitHub

git pull

Atualiza o repositório local com as alterações do GitHub.

Trabalhar com o GitHub

Criar um repositório no GitHub

1 Acede a github.com e faz login
2 Clica no botão "+", opção New repository no canto superior direito
3 Dá um nome ao repositório e escolhe se é público ou privado
4 Podes adicionar um README (ficheiro de apresentação do projeto)
5 Clica em "Create repository"

Ligar um repositório local ao GitHub

Depois de criar um repositório no GitHub, podes conectá-lo ao teu projeto local:

git remote add origin https://github.com/teu_username/nome_repositorio.git
git push -u origin main

Clonar um repositório

Para importar um projeto do GitHub para o teu computador:

git clone https://github.com/username/nome_repositorio.git
Atenção: Certifica-te de que estás na pasta correta quando executas comandos Git. Usa cd "nome_da_pasta" para navegar entre pastas.

Atividade prática guiada

Objetivo: Criar o teu primeiro repositório Git e enviá-lo para o GitHub.

Passo 1: Preparar o Ambiente

1 Cria uma nova pasta para o projeto
2 Abre o terminal (Windows) e acede ao Git Bash
3 Entra na pasta criada anteriormente, copiando o caminho da respetiva pasta:
cd "caminho da pasta"

Passo 2: Inicializar Git

4 Inicializa o repositório Git:
git init

Passo 3: Criar Ficheiro

5 Cria um ficheiro README.md (podes usar um editor de texto)

Conteúdo sugerido: "# O Meu Primeiro Projeto Git"

Passo 4: Primeiro Commit

6 Adiciona o ficheiro:
git add README.md
7 Faz o commit:
git commit -m "Adiciona README inicial"

Passo 5: Enviar para GitHub

8 Cria um repositório no GitHub (como visto na secção anterior)
9 Conecta o repositório local ao GitHub:
git remote add origin URL_DO_TEU_REPOSITORIO
10 Envia para o GitHub:
git branch -M main
git push -u origin main
Parabéns! Criaste o teu primeiro repositório Git e enviaste-o para o GitHub. Agora podes continuar a adicionar ficheiros, fazer commits e enviar alterações.

Desafio Extra

Tenta fazer o seguinte:

  • Cria um novo ficheiro chamado "index.html"
  • Adiciona-o ao Git
  • Faz um commit com uma mensagem apropriada
  • Envia as alterações para o GitHub
